运行环境配置

  • 配置java运行环境变量

    vim /etc/profile
export JAVA_HOME=/home/software/jdk1.8.0_181
export CLASSPATH=$:CLASSPATH:$JAVA_HOME/lib/
export PATH=$PATH:$JAVA_HOME/bin

  • 查看配置是否成功

    java -v

    服务器配置

  • 防火墙配置

    # 关闭防火墙
systemctl stop firewalld.service

# 开放50070端口
firewall-cmd --zone=public --add-port=50070/tcp --permanent

# 关闭50070端口
firewall-cmd --zone=public --remove-port=50070/tcp --permanent

# 刷新配置
firewall-cmd --reload

# 查看当前开放的端口
firewall-cmd --zone=public --list-ports

# 查看防火墙状态
firewall-cmd --state
 
# 查看监听端口
netstat -lnpt
备注：centos7默认没有 netstat 命令，需要安装 net-tools 工具，yum install -y net-tools

  • 网卡IP配置

  • 配置主机名

    vim /etc/sysconfig/network
HOSTNAME=masterA

  • 修改hosts文件

    vim /etc/hosts
192.168.38.133  masterA            #子网地址，最后一位不一样，网关号都是相同的
192.168.38.134  masterB
192.168.38.135  masterC

  • 重启

    reboot

免密登录

ssh-keygen            #生成密钥
ssh-copy-id root@masterA        #密钥拷贝
ssh-copy-id root@masterB
ssh-copy-id root@masterC

配置文件

在配置过程中主要需要对以下几个文件进行配置：core-site.xml、hdfs-site.xml、mapred-site.xml、yarn-site.xml。

  • core-site.xml

    <configuration>
    <property>
        <name>fs.defaultFS</name>            namenode地址
        <value>hdfs://master:9000</value>
    </property>
    <property>
        <name>hadoop.tmp.dir</name>            配置临时目录
        <value>/data/hadoop_repo</value>
   </property>
</configuration>

  • hdfs-site.xml

    <configuration>
<property>
&nbsp;&nbsp;&nbsp;&nbsp;<name>dfs.namenode.secondary.http-address</name>
&nbsp;&nbsp;&nbsp;&nbsp;<value>cloud01:9001</value>
</property>

<property>
&nbsp;&nbsp;&nbsp;&nbsp;<name>dfs.namenode.name.dir</name>
&nbsp;&nbsp;&nbsp;&nbsp;<value>file://${hadoop.tmp.dir}/dfs/name</value>
</property>

<property>
&nbsp;&nbsp;&nbsp;&nbsp;<name>dfs.datanode.data.dir</name>
&nbsp;&nbsp;&nbsp;&nbsp;<value>file://${hadoop.tmp.dir}/dfs/data</value>
</property>

hdfs常见端口设置

参数 描述 默认 配置文件 例子值
fs.default.name namenode namenode RPC交互端口 8020 core-site.xml hdfs://master:8020/
dfs.http.address NameNode web管理端口 50070 hdfs- site.xml 0.0.0.0:50070
dfs.datanode.address datanode　控制端口 50010 hdfs -site.xml 0.0.0.0:50010
dfs.datanode.ipc.address datanode的RPC服务器地址和端口 50020 hdfs-site.xml 0.0.0.0:50020
dfs.datanode.http.address datanode的HTTP服务器和端口 50075 hdfs-site.xml 0.0.0.0:50075

命令操作

  • 启动hdfs文件系统

    start-dfs.sh

  • 关闭hdfs文件系统

    stop-dfs.sh

  • 启动yarn资源管理器

    start-yarn.sh

  • 关闭yarn资源管理器

    stop-yarn.sh

备注：

  • DataNode和NameNode进程同时只能有一个工作。
    1. NameNode在format初始化后会生成clusterID
    2. DataNode在启动后也会生成和NameNode一样的clusterID
    3. 当重新格式化NameNode时，会生成新的clusterID,与未删除DataNode的clusterID不一致

解决方案：在格式化之前，先删除DataNode里面的信息（去temp下删除Data目录）

每次启动时需要将tmp目录下的current清空，以免版本不一致无法启动相关节点。
